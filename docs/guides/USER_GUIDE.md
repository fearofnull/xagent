## 基础使用

### 发送消息

直接在飞书群聊中 @机器人 并发送消息即可：

```
@机器人 你好，请介绍一下自己
```

机器人会使用 Web 管理界面配置的默认 AI 提供商回复你。

### 不使用命令前缀

如果不加任何命令前缀，机器人会使用 Agent（Web 配置的提供商）：

```
@机器人 什么是依赖注入？

@机器人 用 Python 实现一个单例模式
```

机器人会显示具体使用的提供商和模型，例如：`ModelScope (GLM-5)`

## 命令前缀

通过在消息前添加命令前缀，你可以精确控制使用哪个执行器。

### Agent 智能助手（推荐）

Agent 适合快速对话、问答、创意写作、代码生成、多步骤推理、网络搜索、文件操作等场景。

| 命令前缀 | 说明 | 示例 |
|---------|------|------|
| `@agent` | Agent 智能助手 | `@agent 解释一下什么是依赖注入` |
| 无前缀 | 同上（默认行为） | `什么是微服务架构` |

**配置方式**：
1. 访问 Web 管理界面（`http://localhost:8080`）
2. 在"提供商配置"页面添加 AI 提供商（Claude、Gemini、OpenAI、Qwen、ModelScope 等）
3. 设置默认提供商
4. 直接发送消息或使用 `@agent` 命令即可调用

**使用示例**：
```
@机器人 帮我写一个 Python 函数，计算斐波那契数列

@机器人 用 JavaScript 实现一个防抖函数

@机器人 解释一下什么是闭包
```

### CLI 工具（代码操作）

CLI 工具适合代码分析、文件操作、项目结构查看等场景。需要先配置目标项目目录。

| 命令前缀 | AI CLI | 说明 | 示例 |
|---------|--------|------|------|
| `@claude` | Claude CLI | 强大的代码分析能力 | `@claude 分析项目架构` |
| `@gemini` | Gemini CLI | 快速响应 | `@gemini 查看 main.py` |
| `@qwen` | Qwen CLI | 中文优化 | `@qwen 分析代码质量` |

### XAgent 智能助手

XAgent 适合需要多步骤推理、网络搜索、文件操作等复杂任务的场景。

| 命令前缀 | 说明 | 示例 |
|---------|------|------|
| `@agent` | XAgent 智能助手 | `@agent 查看今天的热点新闻` |
| | | `@agent 搜索最新的 Python 3.12 特性` |
| | | `@agent 读取并分析 README.md 文件` |
@机器人 @claude 帮我查看一下 src/main.py 文件的内容

@机器人 @claude 分析一下这个项目的目录结构

@机器人 @gemini 检查代码中是否有潜在的 bug
```

## 会话管理

机器人会自动管理你的对话上下文，让 AI 记住之前的对话内容。

### 查看帮助

```
@机器人 /help
```

显示所有可用命令和使用说明。

### 查看当前会话

```
@机器人 /session
```

显示当前会话的信息：
- 会话 ID
- 消息数量
- 创建时间
- 最后活跃时间

### 创建新会话

```
@机器人 /new
```

创建一个新的会话，清空之前的对话历史。适用于：
- 开始新的话题
- 当前会话消息过多，响应变慢
- 想要重新开始对话

### 查看历史记录

```
@机器人 /history
```

显示当前会话的对话历史（最近 10 条）。

### 会话自动管理

机器人会自动管理会话：
- 单个会话最多保存 50 条消息（可配置）
- 超过限制后自动创建新会话
- 会话闲置 24 小时后自动过期（可配置）

## 会话配置

每个用户或群组可以有独立的配置，支持动态调整。

### 设置项目目录

为 CLI 工具设置目标项目目录：

```
@机器人 /setdir /path/to/your/project
```

**示例**：
```
@机器人 /setdir E:\Projects\my-backend-api

@机器人 /setdir /home/user/projects/frontend-app
```

设置后，CLI 工具将在该目录下执行操作。

### 设置回复语言

设置 AI 回复的语言：

```
@机器人 /lang zh-CN
```

**支持的语言**：
- `zh-CN`: 简体中文
- `en-US`: 英语
- `ja-JP`: 日语
- 其他 ISO 639-1 语言代码

### 设置默认 CLI 提供商

设置默认使用的 CLI 工具：

```
@机器人 /cliprovider claude
```

**可选值**：
- `claude`: Claude CLI
- `gemini`: Gemini CLI
- `qwen`: Qwen CLI

设置后，当启用 AI 意图分类时，系统会使用该 CLI 工具。

### 查看当前配置

```
@机器人 /config
```

显示当前会话的所有配置：
- 项目目录
- 回复语言
- 默认 CLI 提供商

### 重置配置

```
@机器人 /reset
```

重置当前会话的所有配置为默认值。

## 使用场景

### 场景 1: 快速问答

**适合**: 概念解释、代码示例、技术咨询

```
@机器人 什么是 Docker 容器？

@机器人 解释一下 RESTful API 的设计原则

@机器人 如何优化 SQL 查询性能？
```

### 场景 2: 代码编写

**适合**: 算法实现、函数编写、代码片段

```
@机器人 用 Python 写一个二分查找算法

@机器人 用 Java 实现一个线程安全的单例模式

@机器人 用 JavaScript 写一个深拷贝函数
```

### 场景 3: 代码分析

**适合**: 项目分析、代码审查、架构理解

```
@机器人 /setdir /path/to/project

@机器人 @claude 分析一下这个项目的整体架构

@机器人 @claude 查看 src/utils/helper.py 文件，解释它的作用

@机器人 @gemini 检查项目中是否有代码重复
```

### 场景 4: 文件操作

**适合**: 查看文件、理解代码结构

```
@机器人 @claude 读取 README.md 文件的内容

@机器人 @claude 列出 src 目录下的所有 Python 文件

@机器人 @gemini 查看 package.json 的依赖项
```

### 场景 5: 项目理解

**适合**: 新项目上手、代码库探索

```
@机器人 @claude 这个项目是做什么的？主要功能有哪些？

@机器人 @claude 项目的入口文件是哪个？

@机器人 @claude 解释一下项目的目录结构
```

### 场景 6: 多提供商对比

**适合**: 获取不同视角、对比不同方案

通过 Web 管理界面切换不同的提供商，直接发送消息即可：

```
# 在 Web 管理界面切换到 Claude 提供商
@机器人 用 Python 实现一个 LRU 缓存

# 在 Web 管理界面切换到 Gemini 提供商
@机器人 用 Python 实现一个 LRU 缓存

# 在 Web 管理界面切换到 OpenAI 提供商
@机器人 @gpt 用 Python 实现一个 LRU 缓存
```

然后对比不同 AI 给出的实现方案。

### 场景 7: 混合使用

**适合**: 先咨询后实践、理论结合实际

```
# 第一步：咨询设计方案
@机器人 我想重构这个项目的数据库层，有什么建议？

# 第二步：查看现有代码
@机器人 @claude 查看 models/database.py 的当前实现

# 第三步：继续讨论
@机器人 基于现有代码，具体应该怎么重构？
```

### 场景 8: 团队协作

**适合**: 不同项目使用不同配置

```
# 后端团队群组
@机器人 /setdir /team/backend-project
@机器人 /cliprovider claude
@机器人 /lang zh-CN

# 前端团队群组
@机器人 /setdir /team/frontend-project
@机器人 /cliprovider gemini
@机器人 /lang en-US
```

每个群组可以有独立的配置，互不干扰。

## 使用技巧

### 1. 明确你的需求

**好的提问**：
```
@机器人 用 Python 实现一个单例模式，要求线程安全，并解释实现原理
```

**不够好的提问**：
```
@机器人 单例
```

### 2. 提供上下文

**好的提问**：
```
@机器人 @claude 我在 src/utils/cache.py 中实现了一个 LRU 缓存，
但是性能不太好，帮我分析一下可能的原因
```

**不够好的提问**：
```
@机器人 缓存性能不好
```

### 3. 分步骤进行

对于复杂任务，分步骤提问：

```
# 第一步：了解现状
@机器人 @claude 分析项目的测试覆盖率

# 第二步：制定方案
@机器人 基于当前测试覆盖率，如何提高到 80%？

# 第三步：查看具体文件
@机器人 @claude 查看 tests/test_main.py 的内容

# 第四步：继续讨论
@机器人 这个测试文件还缺少哪些测试用例？
```

### 4. 利用会话上下文

机器人会记住对话历史，可以连续提问：

```
你: @机器人 什么是依赖注入？
AI: [解释依赖注入的概念]

你: 能举个 Python 的例子吗？
AI: [提供 Python 示例]

你: 这种方式有什么优缺点？
AI: [分析优缺点]
```

### 5. 使用会话配置

为不同场景配置不同的设置：

```
# 开发 Python 项目
@机器人 /setdir /projects/python-api
@机器人 /cliprovider claude
@机器人 /lang zh-CN

# 开发 JavaScript 项目
@机器人 /setdir /projects/react-app
@机器人 /cliprovider gemini
@机器人 /lang en-US
```

### 6. 善用 Web 管理界面

通过 Web 管理界面（`http://localhost:8080`）：
- 添加和管理 AI 提供商
- 配置 API 密钥和模型参数
- 切换默认提供商
- 查看和编辑会话配置
- 导出和导入配置

## 常见问题

### Q1: 机器人没有回复我？

**可能原因**：
1. 没有 @机器人
2. 机器人不在该群聊中
3. AI 提供商未配置或 API 密钥无效
4. 网络连接问题

**解决方法**：
- 确保消息中 @了机器人
- 检查机器人是否在群聊中
- 访问 Web 管理界面检查提供商配置
- 查看机器人日志确认错误信息

### Q2: CLI 工具命令不工作？

**可能原因**：
1. 未配置目标项目目录
2. 目标目录不存在或无权访问
3. CLI 工具未安装

**解决方法**：
- 使用 `/setdir` 命令设置项目目录
- 确保目录路径正确且可访问
- 安装对应的 CLI 工具（Claude CLI、Gemini CLI 或 Qwen CLI）

### Q3: 如何切换到新话题？

使用 `/new` 命令创建新会话：

```
@机器人 /new
```

### Q4: 机器人回复太慢？

**可能原因**：
1. 会话消息过多，上下文太长
2. 使用了 CLI 工具（比 Agent 慢）
3. AI 服务响应慢

**解决方法**：
- 使用 `/new` 创建新会话
- 对于简单问题使用 Agent（无前缀）
- 等待或稍后重试

### Q5: 如何查看我问了什么？

使用 `/history` 命令：

```
@机器人 /history
```

### Q6: 如何配置 AI 提供商？

访问 Web 管理界面（`http://localhost:8080`）：
1. 登录管理界面
2. 进入"提供商配置"页面
3. 点击"添加提供商"
4. 填写提供商信息（名称、API 密钥、模型等）
5. 设置默认提供商
6. 保存配置

### Q7: Agent 和 CLI 工具有什么区别？

| 特性 | Agent | CLI 工具 |
|------|-------|---------|
| 调用方式 | 无前缀或 `@agent` | `@claude`、`@gemini`、`@qwen` |
| 配置方式 | Web 管理界面 | 环境变量 + `/setdir` 命令 |
| 响应速度 | 快（秒级） | 慢（分钟级） |
| 代码访问 | 无法访问本地代码 | 可访问和操作本地代码 |
| 适合场景 | 问答、翻译、写作、代码生成、工具调用、网络搜索 | 代码分析、文件操作、项目理解 |

### Q8: 如何操作不同的项目？

使用 `/setdir` 命令切换项目目录：

```
# 切换到项目 A
@机器人 /setdir /path/to/project-a
@机器人 @claude 分析项目架构

# 切换到项目 B
@机器人 /setdir /path/to/project-b
@机器人 @claude 查看代码结构
```

### Q9: 会话会保存多久？

- 会话闲置 24 小时后自动过期（可配置）
- 单个会话最多保存 50 条消息（可配置）
- 超过限制后自动创建新会话
- 过期会话会被归档到 `data/archived_sessions/` 目录

### Q10: 如何获得更好的回复？

**提问技巧**：
1. 明确具体：说清楚你想要什么
2. 提供上下文：给出相关背景信息
3. 分步骤：复杂问题分解成小问题
4. 利用历史：在同一会话中连续提问
5. 选择合适的工具：简单问题用 Agent，代码操作用 CLI 工具

### Q11: 如何查看当前使用的是哪个 AI？

机器人的回复会显示具体使用的提供商和模型，例如：

```
来自 ModelScope (GLM-5) 的回复：
[AI 回复内容]
```

你也可以在 Web 管理界面查看当前的默认提供商配置。
